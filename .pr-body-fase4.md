# feat(rubik): render visual experience and banners from config

## Objetivo

Convertir Rubik en un motor que renderiza experiencia visual + pack de banners desde una configuracion unica por cliente, en lugar de mantener HTMLs hardcodeados por slug. Sandhouse Inmobiliaria pasa de placeholder estatico a render dinamico desde la nueva config registry.

- `/dynamic-motion-banner/sandhouse-inmobiliaria/banner-pack/` — overview pack
- `/dynamic-motion-banner/sandhouse-inmobiliaria/banner-vertical.html` — banner vertical 9:16
- `/dynamic-motion-banner/sandhouse-inmobiliaria/banner-horizontal.html` — banner horizontal 16:9

## Renderers creados

- `render-visual-experience.js` — composicion editorial premium, hero + bloques de viaje + CTA. Fondo gradiente intencional, claim serif dominante, CTA dorado, contacto, fallback CSS sin assets.
- `render-banner-pack.js` — overview del pack con claims + ambos formatos embebidos via iframe en modo embed.
- `render-banner-vertical.js` — render 9:16 a 1080x1920 nativos con scaler CSS. Layout: top bar marca/ratio + centro headline serif + chips + bottom contacto/CTA. Diferenciado del horizontal.
- `render-banner-horizontal.js` — render 16:9 a 1920x1080 nativos con scaler. Layout: split 55/45, izquierda branding+headline+CTA, derecha chips+contacto+horario. Diferenciado del vertical.

## Infraestructura compartida

- `_shared/types.js` — JSDoc para `RubikClientConfig`, `RubikVisualExperienceConfig`, `RubikBannerPackConfig`, `RubikMotionTheme`, etc.
- `_shared/normalize-rubik-config.js` — convierte `{visualExperienceSpec, bannerPackSpec, rubikConfigSeed}` (formato AURUM fourHookSpecs) en `RubikClientConfig`. Filtra tokens prohibidos (`Internal draft`, `lorem`, etc), emite warnings estructurados para assets faltantes, garantiza minimo 3 journey blocks.
- `_shared/rubik-config-registry.js` — `getRubikConfigBySlug`, `hasRubikConfig`, `listRubikConfigSlugs`.
- `_shared/preview-router.js` — detecta slug + view desde URL/data-attrs, gestiona modo embed (`?embed=1`), error elegante para slug inexistente, monta badge + QA panel solo fuera de embed.
- `_shared/styles/rubik-preview.css` — sistema de estilos editorial: variables CSS de paleta, fuentes serif/sans/mono, layouts para visual experience, banners 9:16 y 16:9, pack overview, error state, `prefers-reduced-motion`.
