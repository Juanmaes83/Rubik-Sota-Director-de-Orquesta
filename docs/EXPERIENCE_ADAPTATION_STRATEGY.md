# Experience Adaptation Strategy

Rubik Sota Director de Orquesta no es una coleccion de demos. Es una plataforma modular para convertir prototipos visuales, gestuales, IA y hardware en experiencias inmersivas personalizables para sectores, empresas y marcas.

La plataforma debe separar tres niveles:

- **demo tecnica:** prueba que una tecnologia funciona;
- **modulo adaptable:** experiencia con narrativa, privacidad, fallback, marca y salida clara;
- **producto comercial:** modulo estable, medible, personalizable y listo para cliente.

## Que significa adaptar un modulo

Adaptar un modulo significa transformar una prueba visual o tecnica en una experiencia concreta para un sector, una marca y un objetivo comercial.

No basta con cambiar el logo. Una adaptacion correcta debe responder:

- quien lo usa;
- que hace el usuario;
- que ve en pantalla;
- que obtiene al final;
- que datos toca;
- que valor recibe la marca;
- como se despliega;
- como se mide el resultado.

## Que se puede personalizar

Cada modulo adaptable debe declarar que puede cambiarse y hasta donde.

| Area | Personalizacion |
| --- | --- |
| Marca | nombre de campana, tono, identidad visual, copy y posicionamiento |
| Logo | logo principal, marca secundaria, sponsor o co-branding |
| Colores | paleta, contraste, modo evento, modo kiosk o modo premium |
| Assets | imagenes, videos, productos, modelos, fondos, audios y overlays |
| Narrativa | historia, ritual, reto, recorrido, desafio, reveal o souvenir |
| CTA | comprar, reservar, escanear QR, descargar, compartir, pedir cita o desbloquear reward |
| Recompensa | cupon, pase, poster, ranking, badge, producto recomendado o contenido exclusivo |
| Sector | retail, turismo, inmobiliaria, moda, automocion, eventos, educacion, salud o ferias |
| Privacidad | camara opcional, captura local, aviso de permisos, borrado y sin subida por defecto |
| Despliegue | GitHub Pages, Vercel, kiosk, pantalla, tablet, mobile, backend dedicado o hardware |

## Criterios para aceptar un nuevo modulo

Un modulo no entra por acumulacion. Entra si puede convertirse en experiencia adaptable.

| Criterio | Pregunta de decision |
| --- | --- |
| Valor visual | Produce un efecto claro, memorable y entendible en segundos? |
| Valor comercial | Ayuda a vender, captar, explicar, sorprender o convertir? |
| Adaptabilidad sectorial | Puede cambiar de sector sin reescribirlo entero? |
| Peso tecnico | Puede vivir en Rubik o debe quedar como repo externo/engine? |
| Privacidad/camara | Explica permisos, funciona sin camara o tiene fallback? |
| Producto | Tiene salida final, CTA, recompensa o continuidad medible? |

## Ejemplos de adaptacion

### Retail

Un juego gestual puede convertirse en escaparate con producto desbloqueado, cupón, QR y ranking local.

### Turismo

Un oraculo visual puede recomendar rutas, experiencias, hoteles o museos y terminar con itinerario descargable.

### Inmobiliaria

Un before/after o visor inmersivo puede transformar una vivienda en argumento comercial para captacion o venta.

### Moda

Un motor de seleccion gestual puede convertirse en personal shopper, lookbook, sneaker drop o probador visual.

### Automocion

Un modulo 3D o gestual puede mostrar acabados, extras, rutas, configuraciones y CTA a test drive.

### Eventos

Una demo de camara puede convertirse en challenge, ranking, foto recuerdo, sponsor wall y captura de leads.

### Educacion

Un motor interactivo puede convertirse en juego didactico, quiz visual, practica fisica o laboratorio escolar.

### Salud y bienestar

Una experiencia gestual puede guiar postura, movilidad, respiracion, rutina o seguimiento visual sin diagnostico medico.

### Ferias e instalaciones

Un prototipo WOW puede convertirse en atraccion de stand, pantalla de alto impacto, activacion con QR y souvenir.

## Regla de oro

Rubik no debe vender tecnologia cruda. Debe vender experiencias finales entendibles, medibles y adaptables.
