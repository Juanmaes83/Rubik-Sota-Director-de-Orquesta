Rubik Sota Director de Orquesta es una experiencia web interactiva y comercial diseñada para presentar proyectos visuales de forma inmersiva, controlable y memorable.

No es solo un portfolio.
No es solo una landing.
No es solo una demo técnica.

Es una herramienta para dirigir, presentar y vender experiencias visuales usando contenido propio, control gestual, modo cliente, CTA comerciales, presets, exportación de configuración, estadísticas y grabación experimental del escenario principal.

Qué es el proyecto

Rubik Sota Director de Orquesta es una aplicación web en un único archivo HTML que permite crear una presentación visual avanzada e interactiva.

La herramienta permite cargar vídeos o imágenes, organizarlos como proyectos, editarlos comercialmente, añadir llamadas a la acción, activar modo cliente, controlar la experiencia mediante gestos de la mano, cambiar proyectos con swipe, exportar configuraciones, generar estadísticas y grabar clips experimentales del escenario visual.

El proyecto está pensado para funcionar como:

Portfolio interactivo.
Demo comercial.
Presentación experiencial.
Herramienta de venta visual.
Prototipo de producto comercial.
Base para futuras versiones SaaS o plataforma online.
Objetivo principal

El objetivo es transformar una presentación visual tradicional en una experiencia dirigida.

En vez de enseñar simplemente imágenes o vídeos, el usuario puede orquestar una presentación:

cargar contenido;
ordenar proyectos;
editar narrativa;
mostrar CTA;
activar modo cliente;
controlar con gestos;
cambiar de proyecto;
grabar clips;
exportar configuraciones;
medir interacciones.
Para qué sirve

Este proyecto sirve para presentar visualmente proyectos de alto impacto en sectores como:

inmobiliaria premium;
arquitectura;
interiorismo;
diseño;
branding;
publicidad;
inteligencia artificial visual;
producto tecnológico;
experiencias inmersivas;
portfolios creativos;
presentaciones comerciales.

Ejemplo de uso:

Una agencia puede cargar varios vídeos de proyectos, editar el título y CTA de cada uno, activar modo cliente, presentar el contenido en pantalla completa, cambiar de proyecto con la mano y terminar con una llamada a la acción para solicitar una demo.

Concepto: Director de Orquesta

El nombre Director de Orquesta define la función real del proyecto.

La herramienta no se limita a mostrar contenido. Lo dirige.

Cada elemento cumple un papel dentro de la experiencia:

el vídeo o imagen es la pieza principal;
los gestos son la batuta;
la galería es el repertorio;
el modo cliente es el escenario limpio;
los CTA son la conversión;
las estadísticas son la lectura del comportamiento;
la grabación convierte la experiencia en contenido compartible.

El usuario se convierte en director de una presentación comercial visual.

Características principales
Experiencia visual inmersiva

La web presenta el contenido en un escenario visual de alto impacto, con estética oscura, efectos de luz, overlays, microinteracciones y diseño preparado para presentaciones comerciales.

Control gestual

La cámara detecta gestos de la mano mediante MediaPipe Hands.

Gestos principales:

Palma abierta: reproduce vídeo o activa impacto visual en imágenes.
Puño cerrado: pausa el vídeo.
Swipe lateral con mano abierta: cambia de proyecto.
Carga de contenido propio

Permite subir:

vídeos;
imágenes;
reels;
renders;
piezas de portfolio;
material visual de cliente.

Los archivos cargados se guardan localmente en el navegador mediante IndexedDB.

Galería de proyectos

El contenido se organiza en una galería visual. Cada proyecto puede tener título, descripción, sector, CTA y preset comercial asociado.

Modo presentación

Modo limpio para enseñar la experiencia en directo, ocultando parte de la interfaz técnica.

Modo cliente

Modo aún más limpio y comercial. Oculta controles técnicos y muestra una experiencia lista para cliente final.

Editor comercial

Cada proyecto puede editarse con:

título;
subtítulo;
sector;
descripción;
texto CTA;
URL CTA;
orden;
preset aplicado.
CTA seguros

Los CTA pueden apuntar a:

URL web;
WhatsApp;
email;
teléfono.

El sistema bloquea protocolos inseguros como javascript:, data: o file:.

Presets comerciales

Incluye presets para adaptar rápidamente la experiencia a distintos sectores:

Agencia creativa.
Inmobiliaria premium.
Arquitectura.
IA visual.
Producto tecnológico.

También permite guardar presets personalizados.

Exportación e importación

Permite exportar e importar la configuración del proyecto en JSON.

La exportación incluye metadatos, textos, CTA, presets y ajustes, pero no incluye archivos multimedia pesados.

Estadísticas exportables

La experiencia registra métricas como:

palmas detectadas;
puños detectados;
swipes;
comandos;
tiempo de reproducción;
cambios de proyecto;
CTA pulsados;
proyecto más visto.

Estas estadísticas pueden exportarse en formato JSON y texto legible.

Grabación experimental

Permite grabar un clip del escenario principal en formato .webm, con un límite aproximado de 30 segundos.

La grabación es experimental y funciona mejor en escritorio con Chrome o Edge.

El proyecto funciona como una aplicación web autónoma.

No necesita backend para ejecutarse.

El archivo principal es:

index.html

La aplicación usa tecnologías del navegador:

HTML;
CSS;
JavaScript;
IndexedDB;
localStorage;
MediaPipe Hands;
MediaRecorder;
Canvas;
Web APIs nativas.
Modos de uso
1. Modo edición

Es el modo de trabajo.

Permite:

subir contenido;
editar proyectos;
limpiar contenido;
aplicar presets;
exportar configuración;
importar configuración;
exportar estadísticas;
crear clips;
ajustar brillo y volumen.
2. Modo presentación

Modo pensado para enseñar la experiencia sin tanta interfaz.

Permite una visualización más limpia, manteniendo controles básicos.

Atajos:

P = activar/desactivar modo presentación
Esc = salir
3. Modo cliente

Modo comercial final.

Oculta los controles técnicos y deja una experiencia limpia con CTA.

Atajos:

C = activar modo cliente
E = volver a edición
Esc = volver a presentación

También se puede salir del modo cliente desde la píldora superior o mediante el desbloqueo del logo.

Control gestual

La experiencia utiliza la cámara del dispositivo para detectar la mano.

Gestos disponibles
Gesto	Acción
Palma abierta	Reproduce vídeo o activa impacto visual
Puño cerrado	Pausa vídeo
Swipe derecha	Siguiente proyecto
Swipe izquierda	Proyecto anterior
Importante

La cámara solo funciona correctamente en:

localhost

o en una web servida por:

https://

Si abres el archivo directamente con doble clic, algunas funciones pueden fallar por restricciones del navegador.

Carga de contenido propio

La herramienta permite subir archivos desde el ordenador.

Formatos habituales aceptados:

MP4;
WebM;
JPG;
PNG;
WebP;
GIF;
otros formatos compatibles con el navegador.

Los archivos se guardan localmente en el navegador mediante IndexedDB.

Esto significa que:

no se suben a ningún servidor;
no se comparten externamente;
solo existen en el navegador/dispositivo donde se cargan.
Galería de proyectos

Cada archivo subido se añade como proyecto en la galería.

La galería permite:

seleccionar proyecto activo;
cambiar entre proyectos;
mostrar miniaturas;
diferenciar contenido demo y contenido propio;
ordenar los proyectos mediante metadatos;
usar swipe para navegar.
Modo cliente

El modo cliente es una de las partes clave del proyecto.

Está pensado para enseñar la experiencia a un cliente sin que vea controles internos o técnicos.

En modo cliente:

se oculta la navegación técnica;
se oculta el panel de estadísticas;
se oculta la galería;
se oculta el dock comercial;
se muestra el proyecto con CTA;
se mantiene una presentación limpia.
Cómo salir del modo cliente

Opciones:

E = volver a edición
Esc = volver a presentación
clic en píldora de salida
5 clics en el logo

Si la configuración hace referencia a archivos locales que no existen en el nuevo navegador, será necesario volver a subirlos.

Grabación experimental

El sistema incluye una función experimental para grabar el escenario principal.

Características:

duración máxima aproximada: 30 segundos;
formato: .webm;
graba el stage principal;
no graba la webcam;
no garantiza audio;
funciona mejor en Chrome o Edge de escritorio;
en móvil puede no estar disponible.
Limitación importante

La grabación depende de APIs del navegador como MediaRecorder y canvas.captureStream.

No todos los navegadores soportan estas funciones igual.

Limitaciones conocidas
La grabación es experimental.
La grabación no promete audio.
Los archivos multimedia no se incluyen en la exportación JSON.
Los archivos cargados se guardan solo en el navegador actual.
Si se borra la caché o los datos del navegador, se pueden perder proyectos locales.
La detección gestual depende de iluminación, cámara y posición de la mano.
En móviles antiguos puede haber menor rendimiento.
Safari/iOS puede limitar cámara, grabación o reproducción automática.
